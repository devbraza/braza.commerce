## Story

Como sistema, preciso enviar eventos ViewContent e Purchase para o Facebook via Conversion API (server-side) para garantir tracking mesmo com AdBlock/iOS.

## Acceptance Criteria

- [ ] capi.service.ts com metodos sendViewContent(click) e sendPurchase(click, value)
- [ ] POST para https://graph.facebook.com/v21.0/{pixel_id}/events?access_token={token}
- [ ] ViewContent disparado quando click e registrado (E7.3)
- [ ] Purchase disparado quando webhook Yampi confirma compra (E10.1)
- [ ] Payload inclui: event_name, event_time, action_source, user_data (ip, ua, fbc, fbp), custom_data
- [ ] Fire-and-forget: nao bloqueia o fluxo, apenas loga erros
- [ ] Pixel ID e Access Token vem da Campaign ou do Settings (padrao)
- [ ] Se pixel nao configurado, nao dispara (skip silencioso)

## Dev Notes

- Usar fetch nativo do Node (sem dependencia extra)
- Meta CAPI requer access_token com permissao ads_management
- event_time em Unix timestamp (segundos)
- action_source sempre "website"
- Logar resposta da Meta para debug (sucesso/erro)
