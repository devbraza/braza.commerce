## Story
**As an** operator,
**I want** to see how much I'm spending on Facebook Ads and my return on investment directly in the dashboard,
**so that** I can make data-driven decisions about where to invest my ad budget.

## Acceptance Criteria

1. Novo modelo Prisma `AdCostSnapshot`: id, adAccountId, campaignMetaId, campaignName, date (Date), spend (Float), impressions (Int), reach (Int), clicks (Int), cpc (Float), cpm (Float), currency (default "BRL"), syncedAt, userId. Indice unico: [adAccountId, campaignMetaId, date]
2. Novo servico `AdCostService` com metodo `syncCosts(userId, adAccountId, dateRange)` que chama Meta Marketing API `/insights` com paginacao
3. Job cron diario (02:00 UTC) via `node-cron` que sincroniza custos dos ultimos 3 dias para todas as ad accounts conectadas
4. Endpoint `POST /dashboard/sync-costs` para sincronizacao manual com rate limit de 5 minutos
5. Endpoint `GET /dashboard/metrics` estendido com: totalSpend, avgCpc, costPerLead, roas, spendTimeSeries, lastSyncAt, e spend por campanha no topCampaigns
6. Matching de campanhas Meta <-> locais em 2 niveis: vinculo explicito (`Campaign.campaignMetaId`) + heuristica por nome
7. Frontend: 4 novos KPI cards (Gasto Total, CPC Medio, CPL, ROAS) com variacao percentual
8. Frontend: grafico "Gasto vs Receita" com 2 series (requer instalacao de `recharts`)
9. Frontend: coluna "Gasto" na tabela Top Campaigns
10. Frontend: botao "Sincronizar Custos" com loading, toast inline e cooldown de 5 minutos
11. Frontend: indicador de ultima sincronizacao com warning se > 24h
12. Frontend: estado vazio se Facebook nao conectado, com link para Settings

    2. Buscar `User` para obter `accessToken`
    3. Descriptografar token: `this.crypto.decrypt(user.accessToken)` [Source: packages/api/src/meta/meta.service.ts linhas 23-24]
    4. Montar `metaId` com prefixo `act_`: `adAccount.metaId.startsWith('act_') ? adAccount.metaId : 'act_' + adAccount.metaId` [Source: packages/api/src/meta/meta.service.ts linha 83]

      const userId = (req.user as { id: string }).id;
      const lastSync = this.syncCooldowns.get(userId) || 0;

**Notas:**
- `spend`, `cpc`, `cpm` vem como **strings** — converter para `parseFloat()`
- `date_start` = dia do registro — usar para o campo `date` do AdCostSnapshot
- Dados podem atualizar retroativamente por ate 72h (por isso sync dos ultimos 3 dias)
- Rate limit: ~200 calls/hour por ad account
- Permissao `ads_read` ja solicitada no OAuth (Story 1.4)

### Campaign Matching Strategy

```
Nivel 1 (explicito): Campaign.campaignMetaId === insight.campaign_id
  → Match direto, sem ambiguidade

Nivel 2 (heuristica): Se Campaign.campaignMetaId eh null
  → Buscar campanhas locais onde:
    insight.campaign_name.toLowerCase().includes(campaign.name.toLowerCase())
  → Se EXATAMENTE 1 match: auto-setar Campaign.campaignMetaId = insight.campaign_id
  → Se 0 ou 2+ matches: nao linkar (evitar falso positivo)

Fallback: Campanhas Meta sem match exibem com label "(nao vinculada)"
```
