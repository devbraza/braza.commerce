## Story

Como usuario, preciso fazer upload de 1 foto do meu produto como referencia. O sistema gera 6 imagens profissionais estilo e-commerce via Nano Banana Pro (Google Gemini 3 Pro Image).

## Acceptance Criteria

- [ ] POST /pages/:id/reference — upload de 1 imagem de referencia (JPG, PNG, WebP, max 10MB)
- [ ] Foto de referencia salva em /uploads/pages/:pageId/reference.webp (convertida via sharp)
- [ ] Campo referenceImageUrl atualizado na Page
- [ ] POST /pages/:id/generate-images — envia referencia para Nano Banana Pro e gera 6 imagens
- [ ] 6 imagens geradas em estilos e-commerce distintos (ver tabela abaixo)
- [ ] Imagens convertidas para WebP, max 800px largura via sharp
- [ ] Salvas em /uploads/pages/:pageId/1.webp ate 6.webp
- [ ] 6 registros PageImage criados no banco (position 1-6)
- [ ] A foto de referencia NAO aparece no carrossel — so as 6 geradas
- [ ] Validacao MIME type real (magic bytes)
- [ ] Erro 400 se formato invalido ou tamanho > 10MB
- [ ] Erro 500 com mensagem amigavel se Google AI API falhar

## 6 estilos de imagem

| Position | Estilo | Prompt base |
|----------|--------|------------|
| 1 | Hero — fundo branco | Produto centralizado, fundo clean, iluminacao profissional |
| 2 | Angulo 45° | Perspectiva, mostra profundidade e dimensao |
| 3 | Close-up detalhe | Textura, material, acabamento |
| 4 | Lifestyle | Produto em contexto de uso real |
| 5 | Escala/tamanho | Com referencia de tamanho (mao, objeto) |
| 6 | Embalagem/unboxing | Produto embalado, pronto para presente |

## Dev Notes

- SDK: `@google/generative-ai`
- Model: `gemini-3-pro-image-preview` (Nano Banana Pro)
- GOOGLE_AI_API_KEY no .env
- Gerar as 6 imagens em paralelo (Promise.all) para velocidade
- Custo: ~R$4.30 por 6 imagens
- Usar sharp para converter output para WebP e resize
- Rate limit: 5 req/min (prevenir abuso)
