## Story
**As an** operator,
**I want** to generate shipping labels and track order status,
**so that** I have full control of fulfillment.

## Acceptance Criteria

1. Botao "Gerar etiqueta de envio" visivel quando status = `address_complete`
2. Ao clicar, sistema gera etiqueta em formato PDF com: remetente, destinatario, produto, peso estimado
3. Etiqueta abre em nova aba para impressao (PDF via navegador)
4. Campo para registrar codigo de rastreio manualmente (input + botao salvar)
5. Status transitions: `awaiting_address` -> `address_complete` -> `label_generated` -> `shipped` -> `delivered`
6. Ao registrar codigo de rastreio, status atualiza para `shipped`
7. Botao "Marcar como entregue" para transicao `shipped` -> `delivered`
8. Timeline visual no detalhe do pedido mostrando cada transicao de status com data/hora
9. Mensagem de sistema no chat ao mudar status

### Key Technical Details

**Order status flow:**
```
awaiting_address -> address_complete -> label_generated -> shipped -> delivered
```

**Shipping label PDF content:**
- Sender: from user's `shippingData` JSON field (name/company, address, city, state, CEP, phone)
- Recipient: from order fields (fullName, address, city, state, zipCode)
- Product: from related Product model (name)

**System messages:** Saved as Message records with `type=system` and emitted via Socket.io for real-time display.

**User shippingData:** Configured in Settings (Story 6.3), stored as JSON in User model.
