# Story E12.3 — Botao "Publicar com braza.pages" e badge no dashboard

> **Epic:** E12 - braza.commerce → braza.pages Integration
> **Status:** Done
> **Prioridade:** MUST
> **Agente:** @dev (Dex)
> **Depende de:** E12.2
> **PRD:** PRD-002 — FR-01, FR-02

## Story

Como usuario, apos publicar minha pagina no braza.commerce, quero ver um botao "Publicar com braza.pages" na tela de sucesso para publicar em dominio customizado com um clique. No dashboard, quero ver quais paginas ja foram publicadas no braza.pages.

## Acceptance Criteria

### Botao na tela de sucesso (FR-01)

- [ ] Botao "Publicar com braza.pages" aparece no step 4 da criacao de pagina (`pages/new/page.tsx`)
- [ ] Botao aparece ABAIXO dos botoes existentes ("Ver pagina" e "Dashboard") — nao substitui nada
- [ ] Ao clicar: chama `POST /pages/:id/publish-to-braza-pages` via apiFetch
- [ ] Estado "Disponivel": texto "Publicar com braza.pages", botao secundario (outline ou bg sutil)
- [ ] Estado "Enviando": texto "Publicando...", loading spinner, botao desabilitado
- [ ] Estado "Sucesso": texto "Publicado!", URL do braza.pages como link clicavel, cor verde
- [ ] Estado "Erro": texto "Falha ao publicar. Tentar novamente?", cor vermelha, botao de retry
- [ ] Estado "Indisponivel": texto "Servico indisponivel", botao cinza desabilitado (quando 503)
- [ ] Subtexto abaixo do botao: "Publique em dominio customizado com CDN global"
- [ ] Nao quebra layout existente do step 4 — e uma adicao visual abaixo

### Badge no dashboard (FR-02)

- [ ] No dashboard de paginas (`pages/page.tsx`), paginas com `brazaPagesUrl` mostram badge "braza.pages" ao lado do status
- [ ] Badge e clicavel — abre `brazaPagesUrl` em nova aba
- [ ] Cor do badge: diferente do badge de status existente (sugestao: azul ou roxo para diferenciar)
- [ ] Paginas sem `brazaPagesUrl` nao mostram badge — zero mudanca visual para paginas existentes
- [ ] Endpoint `GET /pages` ja retorna `brazaPagesUrl` (campo adicionado no schema em E12.1)
