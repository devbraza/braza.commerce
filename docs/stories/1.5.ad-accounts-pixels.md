## Story
**As an** authenticated user,
**I want** to see my Facebook Ad Accounts and Pixels listed in the system,
**so that** I can choose which accounts to use in my campaigns.

## Acceptance Criteria

1. Endpoint `GET /meta/ad-accounts` busca ad accounts via Meta Marketing API usando o accessToken do usuario
2. Endpoint `GET /meta/ad-accounts/:id/pixels` busca pixels vinculados a uma ad account
3. Ad accounts e pixels sao salvos/atualizados no banco local (cache) apos cada fetch
4. Tela "Ad Accounts" no frontend lista todas as ad accounts com nome, id e status
5. Ao clicar numa ad account, exibe os pixels vinculados
6. Botao "Sincronizar" para forcar re-fetch da Meta API
7. Tratamento de erro se token expirado — exibe mensagem pedindo re-autenticacao
8. Loading states visiveis durante fetch da Meta API

### Key Technical Details

**API Routes (from architecture doc Section 6):**
- `GET /meta/ad-accounts` — JWT auth required
- `GET /meta/ad-accounts/:id/pixels` — JWT auth required

**Prisma Models involved:**
- `AdAccount`: id, metaId, name, status, userId (@@index on userId)
- `Pixel`: id, metaId, name, adAccountId (@@index on adAccountId)

**Meta API endpoints:**
- Ad Accounts: `GET https://graph.facebook.com/v21.0/me/adaccounts?fields=id,name,account_status`
- Pixels: `GET https://graph.facebook.com/v21.0/act_{ad_account_id}/adspixels?fields=id,name`

**Security:** AccessToken must be decrypted from database using CryptoService before calling Meta API. TenantGuard ensures user can only see their own ad accounts.

**Rate limiting:** Standard API rate limit of 60/min/user applies.
