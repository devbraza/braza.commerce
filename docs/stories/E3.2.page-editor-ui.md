# Story E3.2 — Interface de criacao e edicao de pagina

> **Epic:** E3 - Page Builder
> **Status:** Done
> **Prioridade:** MUST
> **Agente:** @dev (Dex)
> **Depende de:** E2.1, E2.2, E3.1
> **PRD:** FR-2.5, FR-2.6, FR-3.3

## Story

Como usuario, preciso de uma interface onde eu faco upload da foto de referencia, informo preco e checkout URL, vejo o conteudo gerado pela IA (6 imagens + textos), edito os textos, e publico a pagina.

## Acceptance Criteria

- [ ] Pagina /pages/new — fluxo de criacao em steps
- [ ] Step 1: Upload de 1 foto de referencia (drag & drop ou click) + campos: preco atual, preco original, URL do checkout
- [ ] Step 2: Botao "Gerar com IA" — loading state enquanto processa (imagens + copy em paralelo)
- [ ] Step 3: Exibe resultado — 6 imagens geradas (grid/carrossel) + campos de texto editaveis (nome, descricao, features, reviews, FAQ)
- [ ] Cada campo de texto tem botao "Regenerar" ao lado
- [ ] Cada imagem tem botao "Regenerar" individual
- [ ] Step 4: Preview da pagina (renderiza template v3.0 com dados atuais)
- [ ] Step 5: Botao "Publicar" — muda status, mostra link publico
- [ ] Pagina /pages/:id/edit — mesma interface para edicao de pagina existente
- [ ] Auto-save rascunho (debounce 2s ao editar textos → PATCH /pages/:id)
- [ ] Responsivo (funciona no mobile)
- [ ] Sem login — acesso direto

## Dev Notes

- Interface simples e direta — NAO e um page builder complexo
- O usuario NAO edita o layout — so os textos e pode regenerar imagens
- Preview usa o render engine (E4.1) ou abre em nova aba
- Usar fonte Inter e cores neutras (consistente com o template)
- Loading state: mostrar skeleton/spinner enquanto IA processa
- Imagens e copy sao gerados em paralelo (2 chamadas API simultaneas)
