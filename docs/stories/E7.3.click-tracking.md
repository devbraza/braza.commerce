## Story

Como sistema, quando um usuario acessa a offer page vinda de um anuncio, preciso capturar o click (fbclid, UTMs, IP) e gerar um click_id unico para rastrear o funil completo.

## Acceptance Criteria

- [ ] POST /tracking/click — registra click (campaignId, fbclid, UTMs) e retorna clickId
- [ ] clickId gerado no formato ck_ + 7 chars aleatorios
- [ ] Captura: fbclid, fbc (calculado), fbp (calculado), IP, user-agent, UTMs
- [ ] Rate limit: 60 req/min por IP
- [ ] Render engine injeta script de tracking na offer page quando pagina tem campanha ativa
- [ ] Script captura fbclid + UTMs da URL e chama /tracking/click
- [ ] Script armazena clickId em variavel para usar no CTA
- [ ] CTA "Comprar agora" redireciona para checkoutUrl com metadata[click_id] e metadata[fbclid]
- [ ] Paginas SEM campanha continuam funcionando normalmente (sem tracking)
- [ ] Evento VIEW_CONTENT registrado automaticamente ao registrar click

## Dev Notes

- fbc = `fb.1.{timestamp}.{fbclid}`
- fbp = `fb.1.{timestamp}.{random}`
- O script de tracking e injetado pelo render.service no template APENAS se campaign_id existe
- O CTA handler intercepta o click e adiciona metadata na URL antes de redirecionar
- Nao bloquear render da pagina se tracking falhar (fire-and-forget)
