## Story
**As an** operator,
**I want** to create and manage my products in the system,
**so that** I can link them to campaigns and generate personalized WhatsApp messages.

## Acceptance Criteria

1. Endpoints REST: `POST /products`, `GET /products`, `GET /products/:id`, `PATCH /products/:id`, `DELETE /products/:id`
2. Campos do produto: name, price, currency (BRL default), whatsappPhone, messageTemplate
3. messageTemplate suporta variavel `{product}` que sera substituida pelo nome do produto
4. Validacao: name obrigatorio, price > 0, whatsappPhone no formato internacional (+5511999999999), messageTemplate obrigatorio
5. Tela "Products" no frontend com listagem em tabela (nome, preco, telefone)
6. Modal/drawer para criar e editar produto com formulario validado
7. Confirmacao antes de deletar produto
8. Produtos vinculados a campanhas nao podem ser deletados (soft block com mensagem)

### Key Technical Details

**Prisma Model:**
```prisma
model Product {
  id              String    @id @default(cuid())
  name            String
  price           Decimal   @db.Decimal(10, 2)
  currency        String    @default("BRL")
  whatsappPhone   String    // +5511999999999
  messageTemplate String    // "Oi, quero saber mais sobre o {product}..."
  userId          String
  user            User      @relation(fields: [userId], references: [id])
  campaigns       Campaign[]
  orders          Order[]
  @@index([userId])
}
```

**API Routes (from architecture doc):** `GET/POST /products` — JWT auth required.

**Validation:** Backend uses class-validator on DTOs, frontend uses Zod schemas.

**Multi-tenant:** TenantGuard + Prisma middleware ensures products are scoped to authenticated user.
