# Story E1.2 — Novo schema do banco de dados

> **Epic:** E1 - Setup

## Story

Como desenvolvedor, preciso criar o novo schema do banco de dados para o braza.commerce com as tabelas Page e PageImage (sem User — auth e v2).

## Acceptance Criteria

- [ ] Enum PageStatus: DRAFT, PUBLISHED, ARCHIVED
- [ ] Model Page: id (cuid), slug (unique), status (PageStatus default DRAFT), title, price (Decimal 10,2), originalPrice (Decimal 10,2), checkoutUrl, referenceImageUrl, aiGeneratedContent (Json), userEditedContent (Json), createdAt, updatedAt, publishedAt
- [ ] Model PageImage: id (cuid), pageId (FK), url, position (Int 1-6), originalName, sizeBytes (Int), createdAt
- [ ] Relacao: Page hasMany PageImages (onDelete: Cascade)
- [ ] Index em PageImage.pageId
- [ ] Migracao Prisma gerada e aplicada com sucesso
- [ ] `npx prisma generate` roda sem erros
- [ ] PrismaService funcional e exportado do common module

## Dev Notes

- Sem model User — todas as paginas sao publicas no MVP
- referenceImageUrl armazena o path da foto original que o usuario subiu
- aiGeneratedContent armazena o JSON completo gerado pelo Claude (copy)
- userEditedContent armazena as edicoes manuais (override parcial)
- Na renderizacao: userEditedContent tem prioridade sobre aiGeneratedContent (merge)
- Campo slug e gerado automaticamente a partir do title via slugify
