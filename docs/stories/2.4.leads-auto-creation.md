## Story
**As an** operator,
**I want** leads to be automatically created from clicks,
**so that** I have visibility of all potential customers from first contact.

## Acceptance Criteria

1. Ao registrar um Click, o sistema cria ou atualiza um Lead vinculado a campanha
2. Lead identificado por phone (extraido quando mensagem WhatsApp chegar) ou por click_id temporariamente
3. Lead contem: phone (nullable ate mensagem chegar), campaignId, productId, clickId, fbclid, ip, userAgent, status (new/contacted/converted/lost), createdAt
4. Endpoint `GET /leads` com filtros: campaignId, productId, status, dateRange
5. Endpoint `GET /leads/:id` retorna lead com clicks e conversations associados
6. Tela "Leads" no frontend com tabela filtravel: nome/telefone, produto, campanha, status, data
7. Lead status atualiza automaticamente: `new` -> `contacted` (quando conversa inicia)
8. Contagem de leads visivel na tela de Campaigns (total por campanha)

### Key Technical Details

**Prisma Model:**
```prisma
model Lead {
  id              String    @id @default(cuid())
  phone           String?   // nullable until WhatsApp msg arrives
  name            String?
  campaignId      String
  productId       String
  clickId         String?   @unique
  fbclid          String?
  status          String    @default("new") // new, contacted, converted, lost, unmatched
  userId          String
  conversations   Conversation[]
  events          Event[]
  orders          Order[]
  @@index([userId])
  @@index([campaignId])
  @@index([phone])
}
```

**API Routes:** `GET /leads` — JWT auth, filtered by TenantGuard.

**Lead lifecycle:**
1. Click registered -> Lead created (status: new, phone: null)
2. WhatsApp message received -> phone populated, conversation created (status: contacted)
3. Purchase event -> status: converted
4. No response -> status: lost (manual or time-based)

**Lead status values:** `new`, `contacted`, `converted`, `lost`, `unmatched`
