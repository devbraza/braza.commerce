## Story

Como usuario, quero ver as metricas de cada landing page (views, clicks, checkouts, compras, revenue) em tempo real pra saber se meus anuncios estao funcionando.

## Contexto Tecnico

**JA EXISTE:**
- `CampaignsService.getStats()` — retorna metricas agregadas (clicks, viewContent, checkouts, purchases, revenue, aov, taxas de conversao)
- Endpoint `GET /pages/:id/stats` (criado na E7.1) — proxy pro CampaignsService

**PRECISA CONSTRUIR:**
- Pagina frontend `/pages/:id/stats` com dashboard visual
- Mini metricas nos cards da listagem de paginas

**Referencia visual:** tracker.braza (sidebar, cards, KPIs, cores escuras, tipografia Inter/JetBrains Mono).

## Acceptance Criteria

### Dashboard (`/pages/:id/stats`)

- [ ] Nova pagina frontend: `packages/web/src/app/pages/[id]/stats/page.tsx`
- [ ] Mostra nome da pagina no topo
- [ ] Filtro por periodo: pills "Hoje | 7d | 30d | Custom"
- [ ] 3 KPIs primarios em cards:
  - Page Views (VIEW_CONTENT count)
  - Checkouts (INITIATE_CHECKOUT count)
  - Compras (PURCHASE count)
- [ ] 3 KPIs secundarios:
  - Revenue (soma dos valores de PURCHASE)
  - Ticket medio (revenue / compras)
  - Conversao geral (compras / page views %)
- [ ] Dados carregados via `GET /pages/:id/stats?from=&to=`
- [ ] Loading state enquanto carrega
- [ ] Estado vazio: mensagem "Nenhum dado ainda" quando tudo e zero

### Listagem de paginas (`/pages`)

- [ ] Cada card de pagina mostra mini metricas: Views | Compras | Revenue
- [ ] Link "Ver metricas" que leva pra `/pages/:id/stats`
- [ ] Dados carregados junto com o `GET /pages` (backend retorna stats basicos)

### Visual

- [ ] Seguir estilo tracker.braza: fundo escuro, cards com glow, KPIs grandes
- [ ] Cores: emerald pra numeros positivos, vermelho pra queda
- [ ] Fonte numeros: JetBrains Mono ou monospace
- [ ] Responsivo: funciona em mobile

## Dev Notes
