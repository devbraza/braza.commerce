## Story
**As a** developer,
**I want** to configure the Z-API webhook to receive WhatsApp messages in the backend,
**so that** the system captures all messages sent by leads.

## Acceptance Criteria

1. Endpoint `POST /webhook/z-api` recebe notificacoes de mensagens da Z-API (JSON direto, sem validacao HMAC)
2. Webhook processa tipos de mensagem: text (campo `text.message`), image (`image.imageUrl`), audio (`audio.audioUrl`), document (`document.documentUrl`)
3. Payload da Z-API mapeado para modelo interno: `phone` → telefone do lead, `messageId` → whatsappMsgId, `momment` → timestamp, `chatName`/`senderName` → nome do lead, `fromMe` → direction (inbound/outbound)
4. Mensagens com `fromMe: true` devem ser descartadas no webhook — ja registradas no envio via Story 3.2, evita duplicacao
5. Ao receber primeira mensagem de um numero, sistema tenta vincular ao Lead existente via click_id extraido do campo `text.message`
6. Se click_id encontrado na mensagem, associa conversation ao lead/click correspondente
7. Se click_id nao encontrado, cria lead orfao com status `unmatched` para triagem manual
8. Mensagem recebida salva no banco: conversationId, content, type (text/image/audio/document), direction (inbound), timestamp, mediaUrl (se midia)
9. Logs estruturados para cada webhook recebido (debug de integracao)
10. Retorna HTTP 200 imediatamente e processa mensagem de forma assincrona (fila Redis) para nao bloquear webhook
11. Na inicializacao do backend, registra a webhook URL na Z-API via `PUT https://api.z-api.io/instances/{instanceId}/token/{token}/update-webhook-received` com header `Client-Token`. Se credenciais nao configuradas (Story 6.3 ainda nao executada), skip silenciosamente com log warning
12. Webhook URL em producao: `https://api.brazachat.shop/webhook/z-api` (URL publica estavel no Railway). Em dev: ngrok ou Cloudflare Tunnel
13. Rate limiter independente no endpoint webhook (separado das rotas autenticadas) — protege contra flood

### Key Technical Details

**Z-API Base URL:** `https://api.z-api.io/instances/{instanceId}/token/{token}/`

**Z-API Authentication:** Header `Client-Token: {clientToken}` em todas as requests.

**Webhook Payload (incoming text message):**
```json
{
  "phone": "5544999999999",
  "fromMe": false,
  "messageId": "A20DA9C0183A2D35A260F53F5D2B9244",
  "momment": 1632228638000,
  "status": "RECEIVED",
  "chatName": "Contact Name",
  "senderName": "Sender Name",
  "isGroup": false,
  "text": {
    "message": "Oi, quero saber mais sobre o Brain Caps ref:ck_A1B2C3D"
  }
}
```

**Webhook Payload (incoming image):**
```json
{
  "phone": "5544999999999",
  "fromMe": false,
  "messageId": "...",
  "momment": 1632228638000,
  "image": {
    "imageUrl": "https://...",
    "caption": "foto do produto",
    "mimeType": "image/jpeg"
  }
}
```

**Media expiry:** Z-API media URLs expiram apos 30 dias.

**Webhook Registration:**
```
PUT https://api.z-api.io/instances/{instanceId}/token/{token}/update-webhook-received
Header: Client-Token: {clientToken}
Body: { "value": "https://api.brazachat.shop/webhook/z-api" }
```

**BullMQ Queue Config:** `whatsapp-messages` — concurrency 10, retry 3x (5s, 30s, 2min).

**Credentials:** Loaded from User settings in database (encrypted with AES-256-GCM via CryptoService). See Story 6.3 for settings implementation.
