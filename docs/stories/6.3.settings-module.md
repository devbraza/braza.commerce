## Story
**As an** operator,
**I want** to configure my account data and integrations in the system,
**so that** the system works correctly with my data and preferences.

## Acceptance Criteria

1. Tela "Settings" com abas: Profile, Integrations, WhatsApp, Tracking, Shipping
2. Aba Profile: nome, email (read-only do Facebook), foto, timezone
3. Aba Integrations: status da conexao Facebook (conectado/desconectado), botao reconectar, lista de permissoes atuais, data da ultima sincronizacao de Ad Accounts
4. Aba WhatsApp — Secao 1 (Credenciais Z-API): Instance ID (texto, por instancia), Token (password, por instancia), Client-Token (password, por conta). Botao "Salvar" persiste credenciais criptografadas (AES-256-GCM) e registra webhook URLs na Z-API automaticamente. Texto de ajuda com passos numerados
5. Aba WhatsApp — Secao 2 (Conexao, visivel apos salvar): Status da conexao (verde Conectado / vermelho Desconectado / amarelo Aguardando QR) via Z-API `GET /status`. QR Code embutido (base64) via `GET /qr-code/image` com auto-refresh 20s e max 3 tentativas. Dados do dispositivo apos conectar (numero, nome, foto, modelo, Business) via `GET /device`. Botao "Reconectar" via `GET /restore-session` com fallback para QR. Botao "Desconectar WhatsApp" com confirmacao
6. Aba Tracking: dominio de tracking (default: `link.brazachat.shop`), preview do link, checkbox UTMs automaticos, utm_source e utm_medium padrao, URL politica de privacidade, indicador SSL (verde/vermelho)
7. Aba Shipping: dados do remetente para etiquetas (nome/empresa, endereco, cidade, estado, CEP, telefone)
8. Endpoint `PATCH /users/settings` para salvar configuracoes (profile, tracking, shipping)
9. Endpoint `PUT /users/whatsapp` para salvar credenciais Z-API (criptografadas com AES-256-GCM)
10. Endpoint `GET /users/whatsapp/status` para verificar status da conexao Z-API
11. Endpoint `GET /users/whatsapp/qrcode` para obter QR Code (base64)
12. Endpoint `GET /users/whatsapp/device` para dados do dispositivo conectado
13. Endpoint `POST /users/whatsapp/restore` para restaurar sessao
14. Endpoint `POST /users/whatsapp/disconnect` para desconectar WhatsApp
15. Dados de remetente usados pela geracao de etiqueta (Story 5.3)
16. Dominio de tracking usado na geracao de links de campanha (Story 2.2)
17. Botao "Desconectar Facebook" com confirmacao (revoga token, mantem dados historicos)
18. Validacao de todos os campos com feedback visual

### Key Technical Details

**API Routes (BrazaChat backend):**
- `PATCH /users/settings` — JWT auth (profile, tracking, shipping)
- `PUT /users/whatsapp` — JWT auth (save Z-API credentials + register webhooks)
- `GET /users/whatsapp/status` — JWT auth (connection status)
- `GET /users/whatsapp/qrcode` — JWT auth (QR Code base64 image)
- `GET /users/whatsapp/device` — JWT auth (connected device info)
- `POST /users/whatsapp/restore` — JWT auth (restore session after disconnect)
- `POST /users/whatsapp/disconnect` — JWT auth (disconnect WhatsApp)

**Z-API Endpoints Used:**

1. Status check:
```
GET https://api.z-api.io/instances/{id}/token/{token}/status
Header: Client-Token: {clientToken}
Response: { "connected": true, "smartphoneConnected": true, "error": "" }
```

2. QR Code (base64 image):
```
GET https://api.z-api.io/instances/{id}/token/{token}/qr-code/image
Header: Client-Token: {clientToken}
Response: base64 encoded image
Note: QR invalidates every 20s, max 3 attempts then regenerate
```

3. Device info:
```
GET https://api.z-api.io/instances/{id}/token/{token}/device
Header: Client-Token: {clientToken}
Response: { phone, name, photo, deviceModel, isBusiness, ... }
```

4. Restore session:
```
GET https://api.z-api.io/instances/{id}/token/{token}/restore-session
Header: Client-Token: {clientToken}
Response: { "value": true }
```

5. Webhook registration (called on save):
```
PUT https://api.z-api.io/instances/{id}/token/{token}/update-webhook-received
Header: Client-Token: {clientToken}
Body: { "value": "https://api.brazachat.shop/webhook/z-api" }

PUT https://api.z-api.io/instances/{id}/token/{token}/update-webhook-delivery
Header: Client-Token: {clientToken}
Body: { "value": "https://api.brazachat.shop/webhook/z-api/delivery" }
```

**Credential Storage:**
```
zapiInstanceId  → stored as plain text (not sensitive)
zapiToken       → encrypted with CryptoService (AES-256-GCM)
zapiClientToken → encrypted with CryptoService (AES-256-GCM)
```

**User Model (updated fields):**
```prisma
model User {
  // ... existing fields ...
  zapiInstanceId   String?   // Z-API instance ID
  zapiToken        String?   // Z-API token (encrypted)
  zapiClientToken  String?   // Z-API client-token (encrypted)
}
```

**Tracking domain:** Default: `link.brazachat.shop`. Used to construct campaign links in Story 2.2.

**Facebook disconnect:** Calls Meta API to revoke token, then clears encrypted tokens from DB. Historical data is preserved.
