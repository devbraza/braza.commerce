## Story

Como sistema, preciso receber webhooks da Yampi quando uma compra e realizada, conectar ao click original via click_id, registrar evento PURCHASE e disparar Meta CAPI.

## Acceptance Criteria

- [ ] POST /api/webhooks/yampi — recebe webhooks da Yampi
- [ ] Validacao HMAC-SHA256 via header X-Yampi-Hmac-SHA256 com secret key
- [ ] Rejeita com 401 se assinatura invalida
- [ ] Processa evento order.paid: extrai metadata[click_id] do payload
- [ ] Conecta compra ao Click original via clickId
- [ ] Registra Event type=PURCHASE com valor do pedido
- [ ] Dispara Purchase para Meta CAPI (E8.1)
- [ ] Processa evento order.created: registra Event type=INITIATE_CHECKOUT
- [ ] Processa transaction.payment.refused: registra Event type=PAYMENT_REFUSED
- [ ] Retorna 200 OK rapidamente (processamento nao bloqueia response)
- [ ] Loga webhooks recebidos para debug

## Dev Notes

- Yampi envia payload com data.metadata contendo os campos que passamos na URL
- Secret key armazenada no Settings (configurada pelo usuario)
- HMAC: crypto.createHmac('sha256', secretKey).update(JSON.stringify(body)).digest('base64')
- Se click_id nao encontrado no payload, ignorar (pode ser compra direta sem tracking)
- Nao falhar se Meta CAPI nao esta configurado — so registra o evento no banco
