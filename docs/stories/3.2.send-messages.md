## Story
**As an** operator,
**I want** to send messages to leads directly from the CRM,
**so that** I can respond without leaving the system.

## Acceptance Criteria

1. Endpoint `POST /conversations/:id/messages` envia mensagem via Z-API: `POST https://api.z-api.io/instances/{instanceId}/token/{token}/send-text` com header `Client-Token`
2. Payload Z-API: `{ "phone": "{leadPhone}", "message": "{content}" }`
3. Mensagem enviada salva no banco com direction `outbound`, status `sent`, e `zaapId` + `messageId` da resposta Z-API
4. Webhook de status update (delivery) via Z-API atualiza status da mensagem no banco
5. Se envio falhar (instancia desconectada, numero invalido), retorna erro descritivo ao frontend
6. Validacao: conversationId deve existir e pertencer ao usuario autenticado
7. Mensagens suportadas no MVP: texto, imagem, audio e documento (Z-API suporta todos nativamente)
8. Upload de midias pelo operador: arquivo enviado para VPS (`/uploads/media/{userId}/{date}/`), servido via endpoint estatico publico, URL passada para Z-API. Limite: 16MB por arquivo (limite WhatsApp)
9. Rate limiting por conversa: maximo 30 mensagens/minuto

### Key Technical Details

**Z-API Send Text:**
```
POST https://api.z-api.io/instances/{instanceId}/token/{token}/send-text
Header: Client-Token: {clientToken}
Body: {
  "phone": "5511999999999",
  "message": "Ola! Obrigado pelo interesse.",
  "delayMessage": 3,
  "delayTyping": 2
}
Response: {
  "zaapId": "...",
  "messageId": "...",
  "id": "..."
}
```

**Z-API Send Image:**
```
POST https://api.z-api.io/instances/{instanceId}/token/{token}/send-image
Body: { "phone": "...", "image": "https://url-da-imagem.jpg", "caption": "Veja o produto" }
```

**Z-API Send Audio:**
```
POST https://api.z-api.io/instances/{instanceId}/token/{token}/send-audio
Body: { "phone": "...", "audio": "https://url-do-audio.ogg" }
```

**Z-API Send Document:**
```
POST https://api.z-api.io/instances/{instanceId}/token/{token}/send-document
Body: { "phone": "...", "document": "https://url-do-doc.pdf", "fileName": "proposta.pdf" }
```

**Prisma Model (updated):**
```prisma
model Message {
  id              String    @id @default(cuid())
  conversationId  String
  content         String
  type            String    @default("text") // text, image, audio, document
  direction       String    // inbound, outbound
  status          String    @default("sent") // sent, delivered, read
  whatsappMsgId   String?   // Z-API messageId
  zaapId          String?   // Z-API internal ID
  mediaUrl        String?   // URL for media messages (images, audio, documents)
  fileName        String?   // Original filename for documents
  @@index([conversationId])
  @@index([createdAt])
}
```

**Delivery webhook registration:**
```
PUT https://api.z-api.io/instances/{instanceId}/token/{token}/update-webhook-delivery
Header: Client-Token: {clientToken}
Body: { "value": "https://api.brazachat.shop/webhook/z-api/delivery" }
```

**Credentials:** Loaded from User settings in database (encrypted). See Story 6.3.
