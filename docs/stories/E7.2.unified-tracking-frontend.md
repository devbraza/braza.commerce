## Story

Como usuario, ao criar minha landing page quero configurar o tracking (URL checkout, Pixel, Access Token) no mesmo fluxo de criacao, sem sair da tela.

## Contexto Tecnico

O frontend hoje tem 3 steps: Upload fotos → Editar textos → Publicar. Precisa adicionar um **Step 3 (Tracking)** entre editar e publicar. O step de publicar vira Step 4 (com slug editavel, ja implementado na E6.1).

**Referencia visual:** tracker.braza (apenas como inspiracao de design — cores, tipografia, inputs).

## Acceptance Criteria

- [ ] Novo Step 3 "Tracking" no fluxo de criacao (`/pages/new`)
- [ ] Step 3 contem 3 campos:
  - URL do checkout (Yampi) — input text, placeholder: `https://seguro.loja.com.br/r/PRODUTO`
  - Pixel ID (Meta) — input text, placeholder: `123456789`
  - Access Token (Meta) — input password (masked), placeholder: `EAAxxxxxxx...`
- [ ] Todos os 3 campos sao **opcionais** — usuario pode pular e publicar sem tracking
- [ ] Botao "Pular" visivel pra ir direto pro Step 4 sem preencher
- [ ] Ao preencher, dados salvos via `PATCH /pages/:id` com os campos de tracking
- [ ] Step 4 (Publicar) continua funcionando como antes (slug editavel + botao publicar)
- [ ] Validacao basica: URL do checkout deve comecar com `https://`
- [ ] Se usuario voltar pro Step 3 depois de preencher, campos mantem os valores

## Dev Notes

- O backend (E7.1) ja aceita checkoutUrl/pixelId/accessToken no DTO
- Usar o mesmo pattern de steps que ja existe no page.tsx
- Inputs seguem o estilo visual do tracker.braza (dark inputs, labels claros)
- Nao precisa de validacao complexa — se o pixel estiver errado, so nao vai rastrear
