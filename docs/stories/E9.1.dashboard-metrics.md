## Story

Como usuario, quero ver um dashboard com metricas do funil de conversao em tempo real, estilo RedTrack.

## Acceptance Criteria

- [ ] GET /campaigns/:id/stats?from=&to= — retorna metricas do funil
- [ ] GET /metrics/overview?from=&to= — metricas globais (todas as campanhas)
- [ ] GET /campaigns/:id/events?page=&limit= — eventos paginados
- [ ] GET /events?page=&limit= — eventos globais paginados
- [ ] Stats retornam: clicks, viewContent, checkouts, purchases, revenue, aov, purchaseRate
- [ ] Frontend /campaigns/:id — dashboard da campanha com:
  - 4 KPI cards primarios (cliques, checkout, compras, ROI) com stat-number animacao
  - 3 KPI cards secundarios (CPA, AOV, revenue)
  - Funil de conversao (barras horizontais com semaforo verde/amarelo/vermelho)
  - Tabela de ultimas conversoes (timestamp, click_id, tipo, valor)
  - Filtro de periodo (pills: Hoje, 7d, 30d, Custom)
- [ ] Frontend /metrics — dashboard geral com mesma estrutura
- [ ] Frontend /events — log de eventos com tabela paginada
- [ ] Layout identico ao tracker.braza (cards, tabelas, badges)
- [ ] Live pulse dot em campanhas com vendas < 5min

## Dev Notes

- Stats calculados com queries COUNT/SUM no Prisma com indices
- Filtro de periodo: from/to como query params (ISO date)
- Funil: percentual entre cada etapa (clicks→viewContent→checkout→purchase)
- Semaforo: purchaseRate >= 3% verde, 1-3% amarelo, < 1% vermelho
- Tabela de eventos: mais recentes primeiro, paginacao 20 por pagina
- Componentes reutilizaveis: KpiCard, FunnelChart, ConversionTable
