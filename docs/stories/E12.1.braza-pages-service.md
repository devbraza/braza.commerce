# Story E12.1 — Serviço de comunicacao com braza.pages

> **Epic:** E12 - braza.commerce → braza.pages Integration
> **Status:** Done
> **Prioridade:** MUST
> **Agente:** @dev (Dex)
> **Depende de:** E11 (braza.pages passthrough — concluido)
> **PRD:** PRD-002 — FR-04, FR-05

## Story

Como sistema, preciso de um servico que encapsule toda comunicacao HTTP com a API do braza.pages, para que o braza.commerce possa publicar paginas no braza.pages de forma confiavel e isolada.

## Acceptance Criteria

- [ ] 3 novos campos opcionais no model Page do Prisma: `brazaPagesUrl` (String?), `brazaPagesDeployId` (String?), `brazaPagesPublishedAt` (DateTime?)
- [ ] Migration Prisma aplicada sem alterar campos existentes
- [ ] Novo modulo NestJS: `BrazaPagesModule` com `BrazaPagesService`
- [ ] `BrazaPagesService.publish(html, slug, domainId)` faz POST para `{BRAZA_PAGES_URL}/api/deploy-passthrough`
- [ ] Headers enviados: `x-api-key` com valor de `BRAZA_PAGES_API_KEY`, `Content-Type: application/json`
- [ ] Body enviado: `{ html, slug, domain_id, source: "braza-commerce" }`
- [ ] Retorna `{ success, url, deployment_id }` em caso de sucesso
- [ ] Trata erros por codigo HTTP: 401 (auth), 400 (json), 413 (tamanho), 422 (validacao), 404 (dominio), 502 (vercel)
- [ ] `BrazaPagesService.healthCheck()` verifica se braza.pages esta acessivel (GET simples com timeout 5s)
- [ ] Configuracao via 3 env vars: `BRAZA_PAGES_URL`, `BRAZA_PAGES_API_KEY`, `BRAZA_PAGES_DEFAULT_DOMAIN_ID`
- [ ] Se env vars nao configuradas, servico nao bloqueia startup — funcionalidade fica desabilitada graciosamente
- [ ] Zero alteracoes em arquivos existentes alem de: `schema.prisma` (+3 campos) e `pages.module.ts` (+1 import)

## Dev Notes

- Contrato completo da API do braza.pages: ver `braza.pages/docs/INTEGRATION-DEPLOY-PASSTHROUGH.md`
- Slug validation no braza.pages: apenas `a-z`, `0-9` e `-`, comeca/termina com letra/numero
- Limite HTML no braza.pages: 50MB (template v3.0 renderizado e ~45KB, muito abaixo)
- `domain_id` no MVP e fixo via env var `BRAZA_PAGES_DEFAULT_DOMAIN_ID`
- API key nunca deve ser exposta no frontend — comunicacao e backend-to-backend
- INTOCAVEIS: `template.html`, `render.service.ts`, todos os endpoints existentes
