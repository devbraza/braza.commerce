## Story

Como sistema, preciso garantir que o webhook da Yampi funcione em producao com a secret key configurada e que eventos duplicados nao sejam processados.

## Contexto Tecnico

O `YampiController` **ja existe e funciona**. Esta story e de **configuracao e hardening**, nao de construcao.

**JA EXISTE:**
- `YampiController` em `src/webhooks/yampi.controller.ts`
- Validacao HMAC-SHA256
- Processamento de order.created (INITIATE_CHECKOUT) e order.paid (PURCHASE)
- Extracao de metadata[click_id]
- CapiService.sendPurchase()

**Webhook ja configurado na Yampi:**
- URL: `https://api.brazachat.shop/api/webhooks/yampi`
- Eventos: order.created + order.paid
- Secret key: valor em env var `YAMPI_WEBHOOK_SECRET` (ja configurada)

## Acceptance Criteria

- [ ] Env var `YAMPI_WEBHOOK_SECRET` configurada no Hetzner (valor ja conhecido)
- [ ] `YampiController` le a secret key da env var (nao do banco Settings)
- [ ] Idempotencia: antes de registrar evento, verificar se ja existe Event com mesmo clickId + type
- [ ] Se evento duplicado, retornar HTTP 200 sem processar (Yampi nao reenvia)
- [ ] Log de warning quando evento duplicado detectado
- [ ] Log de info quando evento processado com sucesso (clickId, type, value)
- [ ] Testar: enviar webhook duplicado → segundo e ignorado
- [ ] Testar: enviar webhook sem click_id → log debug, HTTP 200 (nao quebrar)

## Dev Notes

- A Yampi espera resposta em ate 5 segundos — processamento deve ser rapido
- Hoje o controller ja busca Settings.yampiSecretKey do banco — avaliar se migra pra env var ou mantem banco
- Idempotencia: query simples `SELECT 1 FROM Event WHERE clickId = ? AND type = ?`
- NAO alterar a logica de processamento existente — so adicionar guards
