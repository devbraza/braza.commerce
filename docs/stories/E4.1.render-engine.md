## Story

Como sistema, preciso de um motor de renderizacao que pega os dados da pagina (conteudo da IA + edicoes do usuario + imagens + preco) e gera o HTML final usando o template v3.0.

## Acceptance Criteria

- [ ] render.service.ts recebe Page + PageImages e retorna string HTML completa
- [ ] Template base: docs/prototypes/landing-page-preview.html convertido com placeholders {{campo}}
- [ ] Copia do template em packages/api/src/render/template.html
- [ ] NENHUMA alteracao na estrutura do template — apenas substituicao de conteudo
- [ ] Merge: userEditedContent tem prioridade sobre aiGeneratedContent
- [ ] Campos substituidos: nome, brand, descricao, preco (R$ XX,XX), preco original, desconto (%), parcelamento (3x), features (4), reviews (4), FAQ (3), mini review, checkoutUrl em TODOS os CTAs (3 + sticky)
- [ ] Carrossel: renderiza N imagens (1-6), N dots, N slides — slots sem imagem ficam ocultos
- [ ] Setas do carrossel funcionam com N imagens
- [ ] soldCount base: random entre 150-400
- [ ] Disclaimers no footer renderizados (fixos)
- [ ] Copyright "Braza Publishing Marketing LTDA" (fixo)
- [ ] HTML gerado e self-contained (inline CSS + JS, zero deps exceto Google Fonts async)
- [ ] Tamanho do HTML gerado < 60KB (sem imagens)
- [ ] Todos os elementos dinamicos funcionam: timer, vendidos, toast, estoque, visualizando agora

## Mapeamento template ↔ dados

| Placeholder | Campo de dados |
|------------|---------------|
| {{product_name}} | name |
| {{product_brand}} | brand |
| {{product_description}} | description |
| {{price}} | price (formatado R$ XX,XX) |
| {{original_price}} | originalPrice (formatado) |
| {{discount_pct}} | calculado: Math.round((1 - price/originalPrice) * 100) |
| {{installments}} | calculado: 3x de R$ (price/3).toFixed(2) |
| {{checkout_url}} | checkoutUrl (em todos os CTAs) |
| {{sold_count}} | random 150-400 |
| {{features[0-3]}} | features[] |
| {{reviews[0-3]}} | reviews[] { stars, text, author } |
| {{faq[0-2]}} | faq[] { question, answer } |
| {{mini_review}} | miniReview { initials, stars, text, author } |
| {{carousel_slides}} | PageImages[].url (por position) |
| {{carousel_dots}} | N dots baseado no numero de imagens |

## Dev Notes

- CRITICO: O template v3.0 e INEGOCIAVEL. Renderizacao INJETA dados, nao ALTERA estrutura
- Abordagem: string replacement (sem template engine — SVGs e scripts inline podem quebrar com Handlebars/EJS)
- O desconto e parcelamento sao calculados no servidor
- O original em docs/prototypes/ permanece como referencia visual
- XSS: escapar conteudo da IA antes de injetar (HTML entities)
