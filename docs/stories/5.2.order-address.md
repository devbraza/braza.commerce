## Story
**As an** operator,
**I want** to fill in the order shipping data,
**so that** I can generate the shipping label and dispatch the product.

## Acceptance Criteria

1. Endpoint `PATCH /orders/:id` aceita atualizacao de dados de envio: fullName, address, city, state, zipCode, country
2. Tela de detalhes do pedido acessivel via tela "Orders" ou via link no chat
3. Formulario de endereco com campos: nome completo, endereco, cidade, estado (dropdown UFs), CEP, pais
4. Validacao de CEP: formato XXXXX-XXX para Brasil
5. Ao salvar endereco, status atualiza de `awaiting_address` -> `address_complete`
6. Campos de endereco editaveis enquanto status nao for `shipped` ou `delivered`
7. Botao "Copiar endereco" para clipboard formatado

### Key Technical Details

**API Route:** `GET/PATCH /orders` — JWT auth required.

**Order address fields (from Prisma schema):** fullName, address, city, state, zipCode, country (default "Brasil").

**Status transition:** `awaiting_address` -> `address_complete` (triggered by saving address).

**Brazilian UFs (27 states):** AC, AL, AM, AP, BA, CE, DF, ES, GO, MA, MG, MS, MT, PA, PB, PE, PI, PR, RJ, RN, RO, RR, RS, SC, SE, SP, TO.

**CEP format:** XXXXX-XXX (e.g., 01310-100).

**Validation:** Backend uses class-validator on DTOs, frontend uses Zod schemas.
